Щоб розмістити окремий підкаталог вашого репозиторію на GitHub Pages, ви можете скористатися одним із двох методів: за допомогою налаштувань репозиторію або через спеціальну гілку. Ось поетапні інструкції для кожного з методів:

### Метод 1: Використання `gh-pages` гілки

1. **Створіть окрему гілку**:
   Відкрийте термінал у вашій локальній копії репозиторію і запустіть ці команди:
   ```bash
   git checkout -b gh-pages
   ```

2. **Скопіюйте вміст підкаталогу**:
   Скопіюйте вміст вашого підкаталогу до кореня репозиторію (на цій гілці). Ви можете використовувати команду `cp`:
   ```bash
   cp -r path/to/your/subdirectory/* .
   ```

3. **Застосуйте зміни**:
   Додайте файли до Git і закомітьте:
   ```bash
   git add .
   git commit -m "Deploy subdirectory to GitHub Pages"
   ```

4. **Відправте гілку на GitHub**:
   ```bash
   git push origin gh-pages
   ```

5. **Увімкніть GitHub Pages**:
   - Перейдіть на сторінку вашого репозиторію на GitHub.
   - Увійдіть в налаштування (Settings).
   - Перейдіть до вкладки "Pages".
   - Виберіть гілку `gh-pages` як джерело (Source) та збережіть.

### Метод 2: Використання `docs` підкаталогу

1. **Створіть підкаталог `docs`**:
   Створіть папку `docs` у вашому репозиторії, якщо її ще немає.

2. **Копіюйте вміст підкаталогу**:
   Скопіюйте файли з вашого підкаталогу в нову папку `docs`.

3. **Закомітьте зміни**:
   Знову додайте і закомітьте:
   ```bash
   git add docs
   git commit -m "Add subdirectory content to docs"
   ```

4. **Відправте зміни на GitHub**:
   ```bash
   git push origin main  # або вашу актуальну гілку
   ```

5. **Увімкніть GitHub Pages**:
   - Перейдіть на сторінку вашого репозиторію на GitHub.
   - Увійдіть в налаштування (Settings).
   - Перейдіть до вкладки "Pages".
   - Виберіть `main` (або вашу актуальну гілку) та як джерело `/(root)` і збережіть.

### Поради

- Пам'ятайте, що GitHub Pages підтримує лише статичний контент. Якщо ваш проект використовує динамічний контент, переконайтеся, що він працює у середовищі GitHub Pages.

Використовуючи ці методи, ви зможете розмістити окремий підкаталог вашого репозиторію на GitHub Pages. Якщо у вас є будь-які питання або проблеми, не соромтеся запитувати!
